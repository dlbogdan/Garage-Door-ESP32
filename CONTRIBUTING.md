# Contributing

DIY contributions are welcome. Bug fixes, hardware compatibility improvements,
documentation, tests, and focused features can be proposed through GitHub pull
requests.

## Before submitting

- Open an issue first for large changes or changes affecting safety behavior,
  GPIO ownership, gate movement, authentication, persistence, or OTA updates.
- Keep pull requests focused and explain what changed, why it changed, and how it
  was tested.
- Do not include code, assets, or documentation copied from another project
  unless its license permits inclusion here and you clearly identify its source
  and license.
- Preserve existing third-party notices.
- Run the relevant checks described in [`README.md`](README.md).
